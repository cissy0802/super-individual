① Claude Code 与 Claude.ai：什么时候用哪个Claude Code vs Claude.ai — Where to Draw the Line

ClaudeCLIChat
一句话价值：Claude.ai 是「思考伙伴」，Claude Code 是「动手帮手」。区分清楚，你的产出速度可以翻 3 倍——因为你不再用聊天窗口写代码，也不再用 CLI 做战略讨论。
▎ 推荐工具
  • Claude.ai — 浏览器对话界面，Projects + Artifacts + 文件上传。Pro $20/mo · Max $100-200/mo · claude.ai
  • Claude Code — 终端 CLI，能读写本地文件、跑命令、调 git。含在 Pro/Max 订阅内 或按 API 计费 · claude.com/claude-code
▎ 实战场景

周二早上 9 点，你要做两件事：(1) 跟 PM 讨论 Q3 OKR 怎么排——打开 Claude.ai，创建一个叫「Q3 Planning」的 Project，把团队战略文档拖进去，用 Artifact 反复迭代脑图。(2) 你的副业脚本要把 30 个 CSV 合并去重——打开终端，cd ~/data && claude，让 Claude Code 直接读目录、写脚本、跑、验证。前者要「来回辩论」，后者要「闭嘴干活」。混用是最大的浪费——你不会在 Claude.ai 里复制 30 个文件路径，也不会在 CLI 里讨论团队战略。

▎ Prompt 模板（Claude Code 启动咒语）
# 在项目目录下启动 Claude Code 后，第一条 prompt 永远是「上下文 + 任务 + 约束」

我在 [项目名 / 一句话定位]。
本次任务：[一句话明确目标]。

请先：
1. 用 ls / read 探索 [相关目录]，告诉我现状（不要修改）
2. 列出 2-3 种实现路径与权衡
3. 等我确认后再动手

约束：
- 不引入新依赖除非必要
- 改动 > 50 行前先给 diff 预览
- 测试用 [pytest / vitest / ...]
▎ 动手习题
今晚 30 分钟：在你最近的一个 side project 目录里跑一次 claude，让它读全部代码后用一段话告诉你这个项目「最该重构的 3 个地方」。不要让它改，只让它说。看完关掉。这是体会「读」vs「写」的边界。
▎ 资源

Claude Code 官方文档 · 重点看 CLAUDE.md 与 slash commands 章节

▎ English Insight
agentic coding: AI 能自主读文件、跑命令、迭代修复的编码方式，区别于「补全式 copilot」。"Let it cook": 工程师社群俚语，意为「放手让 Agent 跑完一轮再看」。

② Cursor IDE：把编辑器变成对话Cursor — Editor as Conversation

CursorIDEComposer
一句话价值：Claude Code 适合「让 AI 全权代理」，Cursor 适合「我看着它改、随时拦截」。当你对结果的形状有强意见，Cursor 的 Cmd+K / Composer 让你保持驾驶座。
▎ 推荐工具
  • Cursor — VS Code fork，深度集成 Claude/GPT。Pro $20/mo · Business $40/mo · cursor.com
  • 替代：Zed（更轻量、原生 macOS）· Windsurf（Codeium 出品）· VS Code + GitHub Copilot
▎ 实战场景

你在改一个 React 组件，需要把 prop 重命名并同步到 8 个引用文件。打开 Composer (Cmd+I)，框选这 9 个文件作为 context，告诉它「把 userName 改成 displayName，所有引用同步」。Cursor 给出 diff 视图，你逐文件按 Tab 接受或拒绝。整个过程你看得见每一处改动。这跟 Claude Code 「读完目录直接全改」是两种工作模式——后者效率更高但你失去现场感。原则：当代码是「你的孩子」时用 Cursor，当代码是「一次性脚手架」时用 Claude Code。

▎ Prompt 模板（.cursorrules 文件，放项目根目录）
# .cursorrules — 给 Cursor 的项目宪法，每次对话自动注入

你是 [BigCat] 的 pair programmer。这个项目是 [一句话项目定位]。

技术栈：[TypeScript + React + Tailwind + tRPC]
风格约束：
- 用 const + arrow function，不要 function declaration
- 错误优先返回，不用 try/catch 包整个函数
- 不写 JSDoc 注释除非接口对外

回答约定：
- 改动前先用 1-2 句话说意图
- 大于 30 行的改动先给 plan，等我说"go"再动
- 不要给我「也可以这样做」的备选方案，给我你的最优解
▎ 动手习题
今晚 15 分钟：在 Cursor 里给你最近的项目写一个 .cursorrules。试 3 天，看你按 Tab 接受 diff 的比例是否上升。比例就是你的 prompt 质量分。

③ ChatGPT 你还需要吗？Do You Still Need ChatGPT in 2026?

ChatGPT多模态补位
一句话价值：如果你 80% 时间在 Claude，省下另一个 $20/mo 订阅前请先回答：图像生成、视频生成、网页搜索、Voice Mode、Code Interpreter——这些你一个月用几次？少于 5 次就不值。
▎ 推荐工具（决策清单）
  • ChatGPT Plus — GPT-5 + DALL-E + Sora 短视频 + Advanced Voice + 联网搜索。$20/mo · Pro $200/mo · chatgpt.com
  • 不需要 ChatGPT 的人：写作 + 编码 + 思考为主，且接受 Claude 不联网（用 Perplexity 补）
  • 仍需要的人：需要文生图、文生视频、实时语音陪练、或团队成员在用
▎ 实战场景

BigCat 一个月的实测：Claude.ai 处理 90% 的写作 / 代码 / 思辨。剩下 10% 是——周日陪娃做手抄报需要生成插画（DALL-E 比 Midjourney 快、够用）、出差路上开 Advanced Voice 跟 GPT 练英语口语 20 分钟、研究一只股票要它现场抓最新财报（Claude 不能联网时）。结论：保留 ChatGPT，但降级成「特定场景的瑞士军刀」而不是日常主力。订阅费就当一年 $240 的工具组合权——多模态 + Voice 这两点目前 Claude 还补不上。

▎ Prompt 模板（Voice Mode 英语陪练）
# 打开 ChatGPT app → 右下角耳机图标 → Advanced Voice → 用这个开场

I want to practice English for 15 minutes. Roleplay as a
[senior PM at a Series B startup]. We're meeting because
[I'm pitching an internal tool I built that automates X].

Rules:
1. Interrupt me when my grammar is off — say "Pause" then correct
2. Push back like a real PM: ask about metrics, edge cases, rollout
3. After 15 min, give me 3 phrases I overused and 3 better ones

Ready? Start by asking me the first hard question.
▎ 动手习题
今晚 10 分钟：打开 ChatGPT app 用 Voice Mode 跑一遍上面的 prompt。如果你 30 天没碰 Voice 也没生成过图，下个月退订试试——省下的钱够买半年 Granola。

roleplay prompt: 让模型扮演特定角色；push back: 反对/质疑（职场常用，表示理性反驳）；swiss army knife: 瑞士军刀，比喻多功能但每一项都不是最锋利。

④ 工具切换原则：5 秒决策法The 5-Second Switching Rule

工作流决策框架Raycast
一句话价值：超级个体的瓶颈不是工具不够多，是「切换成本太高」。一个清晰的「该开哪个窗口」决策树，每天省你 30 分钟纠结时间——一年就是一本厚书的阅读量。
▎ 推荐工具（粘合层）
  • Raycast — macOS 系统级唤起器，AI Commands 让任何选中文字一键 LLM 处理。Free · Pro $10/mo · raycast.com
  • Alfred / Spotlight — 替代选项，但 Raycast 的 AI 集成更深
  • BetterTouchTool — 快捷键自定义，把工具切换绑到肌肉记忆
▎ 实战场景（5 秒决策树）

每当你要问 AI 一个问题，问自己 3 个问题，5 秒钟出答案：

Q1：要不要改我磁盘上的文件？ → 是 → Claude Code 或 Cursor（看你想全代理 vs 半代理）

Q2：要不要思考超过 3 个回合？ → 是 → Claude.ai Project（建立长期上下文）

Q3：是 30 秒就完事的碎片问题？ → 是 → Raycast AI Quick Command（不开窗口直接答）

三个都不是 → 默认 Claude.ai 新建对话。BigCat 的关键原则：「永远只在主力工具里建索引」——Claude Projects 是你的思考索引，Obsidian 是你的笔记索引，本地代码库是你的代码索引。AI 工具的对话可以丢，沉淀必须落到三大索引之一。

▎ Prompt 模板（Raycast AI Quick Command）
# Raycast → Extensions → AI Commands → Create Custom AI Command
# 命名「Polish」，绑定到 ⌥+P

You are BigCat's editor. The user will paste a piece of Chinese
or English text. Your job:

1. Fix grammar / typos silently (no explanation)
2. Tighten — cut redundancy, no fluff
3. Keep the voice: direct, technical-but-warm, no corporate jargon
4. If the text is < 30 words, return only the polished version
5. If > 30 words, return polished + ONE line of what you cut

Input:
{selection}
▎ 动手习题
本周末 1 小时：装好 Raycast，把上面的 Polish 命令配好，绑快捷键。然后选中 3 段你最近写的话试一下——如果改完你愿意发出去，这个工作流就成立。这是你 AI 工具栈的第一个「无缝粘合点」。

▎ English Insight
context switching cost: 上下文切换成本（神经科学概念，每次切换约 23 分钟才能完全恢复专注）；glue layer: 粘合层，把异构工具串起来的薄软件层；muscle memory: 肌肉记忆。
